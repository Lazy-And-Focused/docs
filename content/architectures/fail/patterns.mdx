Раздел посвящён неймингу и паттернам, которые Вы должны соблюдать в
рамках кода для Frontend-части приложения.

## Нейминг файлов [#files-naming]

Для именования файлов используется `kebab-case`:

- `example-page/page.tsx`
- `form-login.tsx`
- `users-config.ts`

## Next.js | Layouts & Pages

Функциональные компоненты, экспортируемые по умолчанию, должны иметь
нейминг вида: `ИмяТип`:

```tsx
// home/layout.tsx
type LayoutProps = {
  children: React.ReactNode;
};

export default function HomeLayout({ children }: LayoutProps) {
  return children;
}

// home/page.tsx
export default function HomePage() {
  return <></>;
}

// user/page.tsx
export default function UserPage() {
  return <></>;
}
```
