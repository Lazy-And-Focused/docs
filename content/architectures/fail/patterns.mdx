## Переменные и константы [#vars-and-const]

Этот раздел, скорее всего как и следующий, будет пересказывать
часть из [соглашения команды](../../agreements/general.mdx), но мне
как читателю было бы удобно, чтобы базу накинули и тут. Все спорные
моменты решаются и мы активно обсуждаем их, если находим...

Это было небольшое отступление к тому, что есть вероятность, что в
FAiL есть закономерности, нарушающие основное соглашение, но подобные
моменты мы стараемся сгладить, придти к ком*promise* (за каламбур
извиняюсь)... Ладно, захватите чашу риса, если хотите, ну или суп мису, а мы
продолжаем.

```tsx
// Нейминг констант
const MY_COOL_CONSTANT_BABY = "yeah!";
const API_BACKEND = "https://example.com/api";

// Переменные
const isAdmin = false;
let name = "Failtin"; // каламбур намбер ван

export default function ExampleCatPage({
  params: { id },
}: {
  params: Promise<{ id: string }>;
}) {
  // Стейты и ещё переменные
  const [cat, setCat] = useState<{ name: string } | null>();
  // const themeContext = useContext(ThemeContext);

  // А также функции внутри
  useEffect(() => {
    async function fetchData() {
      const response = await fetch(`${API_BACKEND}/cats/{id}`);
      const data = response.then((res) => res.json());

      setCat(data); // "{ name: "Boris" }" — кому интересно :>
      //                       а ещё кот не мой,
      //                       у мя Перс
    }

    fetchData();
  }, [id]);

  // А вообще лучше использовать обратное условие.
  // Поймёте почему, когда компонент будет чуточку
  // или не на чуточку больше.
  if (cat?.name) {
    return `Meow, ${cat.name}!`;
  }

  return <span>Loading is not supported, lol 😅</span>;
}

// Кстати, если в коде есть ошибки, то не серчайся...
// Я снова более суток не спал и решил внести изменения.
```

После лирического отступления можно радостно продолжить мусолить тему
с паттернами. Что же, тут всё ясно:

1. Избегайте ситуаций, когда самолёт в добавок и дом, и сад, и мяч и
   вертолёт — выносите логику, несвязанную с компонентом напрямую, в
   дочерние файлы
2. Также старайтесь выносить функции из функций, выносить хуки в
   отдельные файлы и тому подобные вещи. Исключением являются те хуки
   или функции, которые не занимают много места на экране
   среднестатистического ноутбука (где-то примерно в районе
   1280-1920px), или когда их мало.
3. Избавляйтесь от дублирующихся компонентов: RedButton, BlueButton,
   WiFiButton, BluetoothButton, BatonButton, BottomBattonButton... Ну вы
   поняли меня... Лучше создать универсальный компонент Button с
   возможностью кастомизации, нежели плодить кучу зомби-вирусов.

## Next.js | Layouts & Pages

Экспортируемые страницы и лейауты должны иметь нейминг вида:
`ИмяТип`:

```js
// Пример для JS, но в команде используется TypeScript.
// Кстати, под него также есть пример, но чуть ниже.

// home/layout.js
export default function HomeLayout({ children }) {
  return children;
}

// home/page.js
export default function HomePage() {
  return <></>;
}

// users/page.js
export default function UserPage() {
  return <></>;
}
```

Но при этом имена для типов props имеют вид: `${Тип}Props`, как в
примере ниже:

```tsx
// users/layout.tsx
type LayoutProps = {
  children: React.ReactNode;
};

export default function UsersLayout({ children }: LayoutProps) {
  return children;
}

// users/[id]/page.tsx
type PageProps = {
  params: Promise<{ id: string }>;
};

export default function UserPage({ params }: PageProps) {
  const { id } = await params;

  return <p>On this page: user with id "{id}"!</p>;
}
```

## Импорты [#imports]

Сортируйте импорты по следующему шаблону:

1. Типы
2. React (если необхходимо)
3. Компоненты
4. Кастомные хуки
5. Контексты и провайдеры
6. Вспомогательные функции или классы
7. Константы, переменные-списки для компонентов
8. Дополнительные данные, такие как стили или JSON

Вот пример:

```ts
// Без понятия, где вы столько каши увидите, но держите...

// Типы
import type { Member } from "@entities/member";
import type { ChatMessage } from "@shared/types";

// React-хуки
import { useState, useEffect } from "react";

// Компоненты
import { ChatWrapper, ChatMessage, UserAvatar } from "@shared/components";
import { Button } from "@shared/components/ui";

import { SendIcon, ProfileIcon } from "@icons";

// Кастомные хуки
import { useChat } from "@shared/lib/channels";

// Контексты и провайдеры
import { ChatProvider } from "@shared/lib/providers";

// Вспомогательная функция
import { parseMessage, getMessages } from "@shared/lib/messages";

// Константы
import { PRIMARY, WARNING } from "@shared/constants/colors";

// Стили и другое
import style from "./example-chat.module.css";
import layoutJson from "./customization/layout.json";
```

## Экспорты [#exports]

Желательно, чтобы у каталога, особенно у вспомогательного
_(утилитарного)_, был `index.ts` с ре-экспортом данных, если это
возможно:

```ts
// мы с вами в example/index.ts

//       example/foo.ts
export * from "./foo.ts";
//       example/bar.ts
export * from "./bar.ts";
```

А также компонент не должен иметь экспорт по умолчанию, если не
является страницей:

```tsx
export function AppHeader() {
  return <header>Hello, World!</header>;
}
// Будет лишним, так как не является страницей:
// export default AppHeader;
```

## Стилизация

Используется модульная CSS-система компонентов при работе с Vanilla
CSS или SASS (SCSS).

При работе с Tailwind CSS и подобными фреймворками или библиотека,
желательно выносить имена классов в константы:

```tsx
import type { HeaderNavLink } from './types';

import { AppHeaderLogotype, AppHeaderSocialLinks } from './ui';
import { DefaultAppHeader, MobileAppHeader } from './variants';

import { WebsiteIcon } from '@icons';

import { useShiftPosition } from './hooks/use-shift-position';
import { useDeviceWidth } from '@shared/hooks';

import clsx from 'clsx';

import { HEADER_SOCIAL_LINKS } from '@shared/lib/constants';

const STYLE = {
  CONTAINER: {
    BASE: [
      'fixed z-50 inset-x-0 top-0',
      'px-4 py-3',
      'grid grid-cols-[6rem_minmax(6rem,_1fr)_6rem]',
      'justify-items-center',
      'border-b-2',
      'transition duration-500 ease-in-out',
    ].join(' '),
    SHIFTED: {
      YES: 'border-primary/75 bg-base-100/20 backdrop-blur-md',
      NO: 'border-transparent',
    },
  },
} as const;

const socialLinks = HEADER_SOCIAL_LINKS.map((link) => ({
  id: link.id,
  href: link.url,
  name: link.customName || link.platform.name,
  icon: link.platform.icon || WebsiteIcon,
}));

export function AppHeader({ links }: { links: HeaderNavLink[] }) {
  const shifted = useShiftPosition();
  const deviceWidth = useDeviceWidth();

  return (
    <header
      className={clsx(
        STYLE.CONTAINER.BASE,
        shifted ? STYLE.CONTAINER.SHIFTED.YES : STYLE.CONTAINER.SHIFTED.NO,
      )}
    >
      <AppHeaderLogotype />

      {!(deviceWidth <= 670) ? (
        <DefaultAppHeader links={links} />
      ) : (
        <MobileAppHeader links={links} />
      )}

      <AppHeaderSocialLinks linkSize={20} links={socialLinks} />
    </header>
  );
};
```
