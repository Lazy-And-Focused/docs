# Организация рутового каталога в FAiL

{/* prettier-ignore */}
> [!CAUTION]
>
> Активно дорабатывается.

## src | Сорс-папка [#src]

{/* prettier-ignore */}
> [!WARNING]
>
> Этой директории может не существовать, если вместо `~/src` каталога
> вы используете рутовую область, либо своё собственное имя.

Папка `~/src` (от англ. "Source") является по умолчанию в FAiL той,
где хранится весь ваш основной код. Этим неким кодом могут быть как
компоненты, так и типы. Другими же словами — основная для работы
папка (рабочая).

## public | Публичные ресурсы [#public]

Публичная директория — все файлы, доступные обычному обывателю, такие
как фавикон сайта или его логотип. Всё верно, эта папка — самая
обычная `public` папка, как в других проекта на React (и не только
React).

Если текст выше не помог разобраться, то внутри каталога хранятся
статичные ресурсы (или же другими словами: "ассеты"), которые не
будут затронуты при сборке. Они также доступны по умолчанию в сети по
корневому URL приложения. Например, файл `~/public/logo.png` будет
доступен по пути: `http://localhost:3000/logo.png` — при условии, что
используются стандартные настройки сервера.

## docs | Документация [#docs]

Каталог приложения... Нет, проекта! Да, каталог проекта, в котором
располагается документация для разработчиков. Обычно там файлы в
формате `.md`, `.mdx`, `.txt` и подобные. Папка спокойно выносится в
вашу проектную вики, если того пожелаете — она не обязательна.
